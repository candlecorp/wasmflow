wick-config::v1

WickConfig

Configuration for Wick applications and components.

Any of the following types:

  • AppConfiguration
  • ComponentConfiguration
  • TypesConfiguration
  • TestConfiguration

LocationReference

A reference to a location on disk, URL, or registry.

Alias for: string

LiquidJsonValue

A liquid JSON template.

Alias for: string

Glob

A glob pattern, ex: **/.html*

Alias for: string

AppConfiguration

The Application configuration defines a standalone Wick application.

Field name Type Description Required? Shortform?
metadata Metadata Associated metadata for this component.
name string The application’s name.
host HostConfig Configuration that controls how this application runs within a host.
import ImportBinding[] Components to import into the application’s scope.
resources ResourceBinding[] Resources that the application can access.
triggers TriggerDefinition[] Configured triggers that drive the application’s behavior.
package PackageDefinition Details about the package for this application.

PackageDefinition

The package details for an application or component.

Field name Type Description Required? Shortform?
files Glob[] The list of files and folders to be included with the package.
registry RegistryDefinition Configuration for publishing the package to a registry. This will be used if the package is published without any additional arguments on the command line. If a tag is specified on the command line, that tag will be used instead.

RegistryDefinition

Field name Type Description Required? Shortform?
registry string The registry to publish to.
namespace string The namespace on the registry. ex: registry.candle.dev//

Metadata

Metadata for the component or application.

Field name Type Description Required? Shortform?
version string The version of the component or application.
authors string[] The authors of the component or application.
vendors string[] Any vendors associated with the component or application.
description string A short description of the component or application.
documentation string Where to find documentation for the component or application.
licenses string[] The license(s) for the component or application.
icon LocationReference The icon for the component or application.

TriggerDefinition

The possible types of resources.

Any of the following types:

  • CliTrigger
  • HttpTrigger
  • TimeTrigger

CliTrigger

A trigger called with a CLI context.

Field name Type Description Required? Shortform?
operation ComponentOperationExpression The operation that will act as the main entrypoint for this trigger. Yes Yes

TimeTrigger

A trigger called with a Time context.

Field name Type Description Required? Shortform?
schedule Schedule Yes
operation ComponentOperationExpression The operation that will act as the main entrypoint for this trigger. Yes Yes
payload OperationInput[] Values passed to the operation as inputs Yes

OperationInput

Field name Type Description Required? Shortform?
name string The name of the operation parameter. Yes
value any The value to pass to the operation parameter. Yes

Schedule

Field name Type Description Required? Shortform?
cron string schedule in cron format with second precision Yes
repeat u16 repeat n times, 0 means forever

ComponentOperationExpression

A reference to an operation with an explicit component definition.

Field name Type Description Required? Shortform?
component ComponentDefinition The component that exports the operation. Yes
name string The operation to call. Yes
with { string : any } Configuration to tie to this operation

HttpTrigger

An HTTP server that delegates to HTTP routers upon requests.

Field name Type Description Required? Shortform?
resource string The TcpPort reference to listen on for connections.
routers HttpRouter[] The HttpRouters that should handle incoming requests

HttpRouter

Any of the following types:

  • RawRouter
  • RestRouter
  • StaticRouter
  • ProxyRouter

ProxyRouter

Field name Type Description Required? Shortform?
path string The path to start serving this router from. Yes
middleware Middleware Middleware operations for this router.
url string The URL resource to proxy to. Yes
strip_path bool Whether or not to strip the router’s path from the proxied request.

RestRouter

Field name Type Description Required? Shortform?
path string The path to start serving this router from. Yes
tools Tools Additional tools and services to enable.
middleware Middleware Middleware operations for this router.
routes Route[] The routes to serve and operations that handle them.
info Info Information about the router to use when generating documentation and other tools.

Tools

Field name Type Description Required? Shortform?
openapi string The path to serve the OpenAPI spec from

Info

Information about the router to use when generating documentation and other tools.

Field name Type Description Required? Shortform?
title string The title of the API.
description string A short description of the API.
tos string The terms of service for the API.
contact Contact The contact information for the API.
license License The license information for the API.
version string The version of the API.
documentation Documentation The URL to the API’s terms of service.

Documentation

Documentation information for the API.

Field name Type Description Required? Shortform?
url string The URL to the API’s documentation.
description string A short description of the documentation.

License

The license information for the API.

Field name Type Description Required? Shortform?
name string The name of the license.
url string The URL to the license.

Contact

The contact information for the API.

Field name Type Description Required? Shortform?
name string The name of the contact.
url string The URL to the contact.
email string The email address of the contact.

Route

A route to serve and the operation that handles it.

Field name Type Description Required? Shortform?
name string The name of the route, used for documentation and tooling.
methods string[] The HTTP methods to serve this route for.
uri string The path to serve this route from.
operation ComponentOperationExpression The operation that will act as the main entrypoint for this route. Yes Yes
description string A short description of the route.
summary string A longer description of the route.

StaticRouter

Field name Type Description Required? Shortform?
path string The path to start serving this router from. Yes
middleware Middleware Middleware operations for this router.
volume string The volume to serve static files from. Yes
fallback string Fallback path (relative to volume resource) for files to serve in case of a 404. Useful for SPA’s. if volume resource is: /www and fallback: index.html, then a 404 will serve /www/index.html

RawRouter

A RawHttpRouter delegates raw requests and bodies to operations based on the request path.

Field name Type Description Required? Shortform?
path string The path to start serving this router from. Yes
middleware Middleware Middleware operations for this router.
codec Codec The codec to use when encoding/decoding data.
operation ComponentOperationExpression The operation that handles HTTP requests. Yes Yes

Middleware

Request and response operations that run before and after the main operation.

Field name Type Description Required? Shortform?
request ComponentOperationExpression[] The middleware to apply to requests.
response ComponentOperationExpression[] The middleware to apply to responses.

ResourceDefinition

The possible types of resources.

Any of the following types:

  • TcpPort
  • UdpPort
  • Url
  • Volume

TcpPort

A TCP port to bind to.

Field name Type Description Required? Shortform?

address string The address to bind to.

Volume

A filesystem or network volume resource.

Field name Type Description Required? Shortform?
path string The path. Yes

UdpPort

A UDP port to bind to.

Field name Type Description Required? Shortform?
port u16 The port to bind to.
address string The address to bind to.

Url

A URL configured as a resource.

Field name Type Description Required? Shortform?
url string The url string. Yes

TypesConfiguration

A type definition for a Wick Components and Operations

Field name Type Description Required? Shortform?
name string The name of this type.

types TypeDefinition[] Additional types to export and make available to the type.

TestConfiguration

A configuration for a Wick Component

Field name Type Description Required? Shortform?

ComponentConfiguration

A configuration for a Wick Component

Field name Type Description Required? Shortform?
name string The name of this component.
metadata Metadata Associated metadata for this component.
host HostConfig Configuration for when wick hosts this component as a service.
resources ResourceBinding[] Resources that the application can access.
import ImportBinding[] Components or types to import into the application’s scope.
types TypeDefinition[] Additional types to export and make available to the component.
requires BoundInterface[] Interfaces the component requires to operate.
labels { string : string } The labels and values that apply to this manifest.
component ComponentKind Configuration specific to different kinds of components. Yes
tests TestConfiguration[] Assertions that can be run against the component to validate its behavior.
package PackageDefinition Details about the package for this component.

HostConfig

Host configuration options.

Field name Type Description Required? Shortform?
allow_latest bool Whether or not to allow the :latest tag on remote artifacts.
insecure_registries string[] A list of registries to connect to insecurely (over HTTP vs HTTPS).
timeout u64 The timeout for network requests (in ms).
rpc HttpConfig Configuration for the GRPC server.

HttpConfig

Configuration for the GRPC service.

Field name Type Description Required? Shortform?

pem LocationReference Path to pem file for TLS.
key LocationReference Path to key file for TLS.
ca LocationReference Path to CA file.

GrpcUrlComponent

ManifestComponent

Field name Type Description Required? Shortform?

CompositeOperationDefinition

A definition for a single composite operation.

Field name Type Description Required? Shortform?
name string The name of the operation.
with Field[] Any configuration required by the operation.
inputs Field[] Types of the inputs to the operation.
outputs Field[] Types of the outputs to the operation.
components string[] A list of components the schematic can use.
uses InstanceBinding[] A map of IDs to specific operations.
flow FlowExpression[] A list of connections from operation to operation.
operations CompositeOperationDefinition[] Additional flow operations scoped to this operation.

FlowExpression

A flow operation, i.e. a connection from one operation to another.

Any of the following types:

  • ConnectionDefinition
  • BlockExpression

BlockExpression

A list of FlowExpressions

Field name Type Description Required? Shortform?
expressions FlowExpression[] Yes

ConnectionDefinition

A connection between Operations and their ports. This can be specified in short-form syntax (where applicable).

Field name Type Description Required? Shortform?
from ConnectionTargetDefinition The upstream operation port. Yes
to ConnectionTargetDefinition The downstream operation port. Yes

ConnectionTargetDefinition

A connection target e.g. a port on a reference. This can be specified in short-form syntax (where applicable).

Field name Type Description Required? Shortform?
instance string The instance ID of the operation. Yes
port string The operation port. Yes
data { string : any } The default value to provide on this connection in the event of an error.

Field name Type Description Required? Shortform?
name string The name of the binding. Yes

TestDefinition

A test case for a component.

Field name Type Description Required? Shortform?
name string The name of the test. Yes
operation string The operaton to test. Yes
inherent InherentData Inherent data to use for the test.
with { string : any } The configuration for the operation, if any.
input PacketData[] The inputs to the test.
output PacketData[] The expected outputs of the operation.

InherentData

Data inherent to transactions.

Field name Type Description Required? Shortform?
seed u64 An RNG seed.
timestamp u64 A timestamp.

PacketData

Either a success packet or an error packet.

Any of the following types:

  • PayloadData
  • ErrorData

PayloadData

A simplified representation of a Wick data packet & payload, used to write tests.

Field name Type Description Required? Shortform?
name string The name of the port to send the data to. Yes
flags PacketFlags Any flags set on the packet.
data any The data to send.

ErrorData

Field name Type Description Required? Shortform?
name string The name of the port to send the data to. Yes
flags PacketFlags Any flags set on the packet.
error string The error message. Yes

PacketFlags

Flags set on a packet.

Field name Type Description Required? Shortform?
done bool When set, indicates the port should be considered closed.
open bool When set, indicates the opening of a new substream context within the parent stream.
close bool When set, indicates the closing of a substream context within the parent stream.
